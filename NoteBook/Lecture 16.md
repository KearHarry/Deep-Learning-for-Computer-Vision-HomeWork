# Lecture 16：图像分割 (Image Segmentation)

---

## 📌 概览

本讲介绍像素级别的视觉理解任务——图像分割：

1. **语义分割** — 每个像素分配一个类别
2. **实例分割** — 区分同类的不同实例
3. **全卷积网络 (FCN)** — 语义分割的开山之作
4. **上采样方法** — 从低分辨率恢复空间细节

---

## 第一部分：分割任务定义

### 1.1 三种分割任务

| 任务 | 输出 | 区分实例？ | 例子 |
| ---- | ---- | ---------- | ---- |
| 语义分割 | 每像素类别标签 | ❌ | 所有"人"是同一类 |
| 实例分割 | 每像素类别 + 实例ID | ✅ | 不同的"人"是不同实例 |
| 全景分割 | 语义 + 实例 | ✅ | stuff（天空）+ things（人） |

### 1.2 语义分割的挑战

- 需要**像素级**的精确预测
- 需要同时理解**全局语义**（这是什么）和**局部细节**（边界在哪）
- 训练数据标注成本高

---

## 第二部分：语义分割方法

### 2.1 朴素方法：滑动窗口（❌ 极慢）

对每个像素取一个小 patch 送入分类器。问题：大量重复计算，不可行。

### 2.2 全卷积网络 FCN（⭐ 里程碑）

> 核心思想：把分类网络的全连接层替换为卷积层，输出与输入同尺寸的预测图。

```
输入图像 (3, H, W)
  → CNN backbone 下采样 → 低分辨率特征图 (C, H/32, W/32)
  → 1×1 Conv → 类别分数图 (num_classes, H/32, W/32)
  → 上采样 → 预测图 (num_classes, H, W)
  → 逐像素 argmax → 分割结果
```

**损失函数**：逐像素的交叉熵损失

$$L = -\frac{1}{HW}\sum_{i,j} \log P(y_{ij} | x)$$

### 2.3 上采样方法（⭐ 重点）

#### 双线性插值 (Bilinear Interpolation)

用周围 4 个像素的加权平均计算新像素值。简单高效，但不可学习。

#### 转置卷积 (Transposed Convolution)

也称反卷积 (Deconvolution)，是**可学习的上采样**：

```
输入: (C, H, W)
转置卷积 (stride=2): (C', 2H, 2W)
```

- 相当于在输入元素之间插入零，然后做普通卷积
- 参数可学习，能学到比双线性插值更好的上采样

**棋盘格伪影** (Checkerboard Artifacts)：当 kernel size 不能被 stride 整除时，转置卷积可能产生棋盘格状的伪影。解决方案：使用 resize + convolution 代替。

#### Max Unpooling

利用前向传播时 Max Pooling 记录的最大值位置，将值放回原位，其余补零。

### 2.4 U-Net 架构（⭐⭐ 经典）

```
编码器路径（下采样）          解码器路径（上采样）
  Conv → Pool ─────────→ Concat → Conv → 上采样
  Conv → Pool ─────────→ Concat → Conv → 上采样
  Conv → Pool ─────────→ Concat → Conv → 上采样
       └── Bottleneck ──┘
```

**Skip Connections 的作用**：
- 低层特征保留空间细节（边缘、纹理）
- 高层特征提供语义信息
- 跳跃连接融合两者，实现精确的分割边界

### 2.5 空洞卷积 (Dilated/Atrous Convolution)

在卷积核元素之间插入空洞，**不增加参数量但扩大感受野**：

- Dilation rate = 1：标准 3×3 卷积，感受野 3×3
- Dilation rate = 2：等效感受野 5×5
- Dilation rate = 4：等效感受野 9×9

**用途**：在不降低分辨率的情况下获得大感受野，常用于 DeepLab 系列。

## 第三部分：实例分割

### 3.1 Mask R-CNN（⭐ 重要）

在 Faster R-CNN 基础上，给每个 ROI 多加一个分支预测像素级掩码：

```
Faster R-CNN 的 ROI 特征
  → 分类头 → 类别
  → 框回归头 → 精修框
  → 掩码头 → 每个类的二值掩码 (mask_size × mask_size)
```

**关键设计**：
- 掩码预测是**类别无关的**（每个类独立预测一个二值掩码）
- 使用 **ROI Align**（不是 ROI Pooling）避免量化误差
- 掩码损失只在 GT 类别对应的掩码上计算

---

## 💡 重点总结

1. **FCN 开创了端到端语义分割**：全卷积 + 上采样
2. **编码器-解码器结构** (U-Net) 是分割网络的主流设计
3. **Skip connections** 融合低层细节和高层语义，产生精确边界
4. **转置卷积**是可学习的上采样，但需注意棋盘格伪影
5. **空洞卷积**在不降低分辨率的情况下扩大感受野
6. **Mask R-CNN = Faster R-CNN + 掩码分支**，实现实例级分割
