# Lecture 21：自监督学习 (Self-Supervised Learning)

---

## 📌 概览

本讲介绍不依赖人工标注的表示学习方法：

1. **自监督学习的动机** — 标注数据昂贵，无标注数据廉价
2. **前置任务 (Pretext Tasks)** — 设计巧妙的伪任务来学习特征
3. **对比学习 (Contrastive Learning)** — SimCLR、MoCo
4. **与迁移学习的关系** — 预训练范式的演变

---

## 第一部分：为什么需要自监督学习？

### 1.1 标注数据的瓶颈

- ImageNet：1400万张标注图像，花费数年
- 真实世界中：绝大多数数据没有标注
- **自监督学习**：从数据本身自动生成监督信号

### 1.2 核心思想

> 设计一种任务，让模型不需要人工标签就能学到有用的视觉表示。

---

## 第二部分：前置任务 (Pretext Tasks)

### 2.1 常见前置任务

| 任务 | 做法 | 学到什么 |
| ---- | ---- | -------- |
| 图像旋转预测 | 将图像旋转 0°/90°/180°/270°，预测角度 | 物体朝向感知 |
| 拼图 (Jigsaw) | 将图像切块打乱，预测排列顺序 | 空间关系理解 |
| 着色 (Colorization) | 输入灰度图，预测彩色 | 语义理解 |
| 补全 (Inpainting) | 遮挡部分区域，预测被遮挡内容 | 上下文推理 |
| 相对位置预测 | 给两个 patch，预测相对位置 | 空间关系 |

### 2.2 前置任务的局限

- 学到的特征可能过于针对前置任务本身
- 需要精心设计，不同任务效果差异大
- 不够通用

## 第三部分：对比学习（⭐⭐ 核心）

### 3.1 核心思想

> 相似的样本（正样本对）在特征空间中靠近，不相似的样本（负样本对）远离。

### 3.2 SimCLR 框架

```
原始图像 x
  → 随机增强变换1 → x_i ┐
  → 随机增强变换2 → x_j ┘  (正样本对)
  
  → CNN Encoder → h_i, h_j
  → Projection Head (MLP) → z_i, z_j
  → 对比损失 (NT-Xent)
```

**NT-Xent 损失**：

$$L_{i,j} = -\log \frac{\exp(\text{sim}(z_i, z_j) / \tau)}{\sum_{k \neq i} \exp(\text{sim}(z_i, z_k) / \tau)}$$

- $\text{sim}$：余弦相似度
- $\tau$：温度参数
- 分子：正样本对的相似度
- 分母：正样本 + 所有负样本的相似度之和

### 3.3 数据增强是关键

SimCLR 中最重要的增强组合：
1. 随机裁剪 + Resize
2. 随机颜色失真
3. 随机高斯模糊

> 强的数据增强迫使网络学习**语义不变**的特征，而非颜色/纹理等表面特征。

### 3.4 MoCo (Momentum Contrast)

**SimCLR 的问题**：需要非常大的 batch size（4096+）来获得足够多的负样本。

**MoCo 的解决方案**：维护一个动量更新的**队列**作为负样本库。

```
Query Encoder: 正常更新
Key Encoder: 动量更新 (θ_k ← m·θ_k + (1-m)·θ_q)
Queue: 存储最近的 K 个 key 作为负样本
```

- 不需要超大 batch size
- 动量更新让 key encoder 平滑变化，保持队列中特征的一致性

### 3.5 BYOL / SimSiam：不需要负样本？

最新的方法表明，通过适当的架构设计（如 stop-gradient），甚至**不需要负样本**也能学到好的表示，避免了模式坍塌。

---

## 💡 重点总结

1. **自监督学习的核心**：利用数据本身的结构作为监督信号，无需人工标注
2. **对比学习**是目前最成功的自监督方法：拉近正样本、推开负样本
3. **数据增强的选择**比具体的损失函数更重要
4. **SimCLR 需要大 batch，MoCo 用队列解决**负样本数量问题
5. 自监督预训练的特征质量**逼近甚至超过**有监督 ImageNet 预训练
