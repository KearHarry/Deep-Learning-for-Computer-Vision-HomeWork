# Lecture 3：线性分类器

---

## 📌 概览

本讲从 KNN 的局限性出发，引入线性分类器，从三个视角理解它：

1. **代数视角** — 矩阵乘法 $f(x) = Wx + b$
2. **视觉视角** — 每一类的权重可视化为"模板"
3. **几何视角** — 每一类对应一个超平面

并引出两种损失函数：**SVM Loss** 和 **Softmax Loss**

---

## 第一部分：从 KNN 到线性分类器

### 1.1 KNN 的问题

| 问题 | 说明 |
| ---- | ---- |
| 预测慢 | 每次预测都需要与所有训练数据比较，$O(N)$ |
| 存储大 | 必须保存全部训练数据 |
| 像素距离无意义 | 像素级别的距离无法捕捉语义相似性 |

> **理想模型**：训练慢没关系，但预测必须快。线性分类器就是这样——训练时学出权重 $W$，预测时只需一次矩阵乘法。

### 1.2 参数化方法（⭐ 核心思想）

$$f(x, W) = Wx + b$$

- $x$：输入图像，展平为向量 $(D \times 1)$，如 $32 \times 32 \times 3 = 3072$
- $W$：权重矩阵 $(C \times D)$，$C$ 为类别数
- $b$：偏置向量 $(C \times 1)$
- $f$：输出分数向量 $(C \times 1)$，每个类一个分数

> **Bias Trick**：可以把 $b$ 合并进 $W$，在 $x$ 末尾加一个 $1$：
>
> $$f(x, W) = W \begin{bmatrix} x \\ 1 \end{bmatrix}$$
>
> 这样就只有一个参数矩阵需要学习。

## 第二部分：三种视角理解线性分类器

### 2.1 代数视角

$$\begin{bmatrix} s_{\text{cat}} \\ s_{\text{dog}} \\ s_{\text{ship}} \end{bmatrix} = \begin{bmatrix} W_1 \\ W_2 \\ W_3 \end{bmatrix} \cdot x + \begin{bmatrix} b_1 \\ b_2 \\ b_3 \end{bmatrix}$$

每个类别的分数 = 该类的权重向量与输入的**点积** + 偏置。

### 2.2 视觉视角（⭐ 直观理解）

把 $W$ 的每一行 reshape 回图像的形状，就可以可视化为一个"模板"：

- **猫类的权重**看起来像一个模糊的猫脸
- **汽车类的权重**看起来像一个模糊的汽车轮廓

> **局限性**：每个类别只有**一个模板**。如果一类物体有多种外观（如左看的马和右看的马），线性分类器会把它们"平均"成一个混合模板，效果很差。

### 2.3 几何视角

在高维空间中，每个类别的权重向量 $w_i$ 定义了一个**超平面**（决策边界）：

- 超平面一侧的点得分为正（属于该类）
- 另一侧得分为负（不属于该类）

> **本质限制**：线性分类器只能画**直线**（超平面）来分割数据。对于不能被直线分开的数据（如 XOR 问题、环形数据），线性分类器无能为力。

## 第三部分：损失函数

### 3.1 什么是损失函数？

损失函数衡量的是：**模型的预测有多"烂"**。

$$L = \frac{1}{N} \sum_{i=1}^{N} L_i(f(x_i, W), y_i) + \lambda R(W)$$

- **数据损失**：预测与真实标签的差距
- **正则化项**：防止模型过拟合，偏好"简单"的权重

### 3.2 多类 SVM 损失 / Hinge Loss（⭐ 重点）

$$L_i = \sum_{j \neq y_i} \max(0, \; s_j - s_{y_i} + \Delta)$$

- $s_j = f(x_i, W)_j$：第 $j$ 类的分数
- $s_{y_i}$：正确类的分数
- $\Delta$：安全边际（margin），通常取 $1$

**直观理解**：正确类的分数至少要比每个错误类的分数高出 $\Delta$，否则就会产生损失。

> **举个例子**：
>
> 假设 $s = [3.2, 5.1, -1.7]$，正确类是第 0 类（$s_{y_i} = 3.2$），$\Delta = 1$
>
> - $j=1$：$\max(0, 5.1 - 3.2 + 1) = \max(0, 2.9) = 2.9$ ✗ 有损失
> - $j=2$：$\max(0, -1.7 - 3.2 + 1) = \max(0, -3.9) = 0$ ✓ 无损失
> - $L_i = 2.9$

### 3.3 Softmax / 交叉熵损失（⭐ 重点）

**Step 1**：将分数转为概率（Softmax 函数）

$$P(y = k \mid x) = \frac{e^{s_k}}{\sum_j e^{s_j}}$$

**Step 2**：计算交叉熵损失

$$L_i = -\log P(y_i \mid x_i) = -\log \frac{e^{s_{y_i}}}{\sum_j e^{s_j}}$$

**直观理解**：正确类的概率越高，$-\log$ 后的值越小，损失越低。

> **数值稳定性**（⭐ 易出 bug）：
>
> 直接计算 $e^{s_k}$ 可能溢出。技巧是先减去最大值：
>
> $$\frac{e^{s_k}}{\sum_j e^{s_j}} = \frac{e^{s_k - \max(s)}}{\sum_j e^{s_j - \max(s)}}$$

### 3.4 SVM vs Softmax 对比

| 特性 | SVM (Hinge) | Softmax (Cross-Entropy) |
| ---- | ----------- | ----------------------- |
| 关注点 | 只关心正确类是否比错误类高出 margin | 关心所有类的概率分布 |
| 分数解释 | 无概率含义 | 概率分布（归一化） |
| 损失特性 | 一旦满足 margin 就不再优化 | 永远在"推"正确类概率趋近 1 |
| 梯度稀疏性 | 稀疏（只有违规类有梯度） | 密集（所有类都有梯度） |

## 第四部分：正则化

### 4.1 为什么需要正则化？

如果只有数据损失 $L_{\text{data}}$，可能存在无穷多组 $W$ 都能使损失为 0。正则化帮助我们选择"更好"的那一个。

### 4.2 常见正则化

| 类型 | 公式 | 偏好 |
| ---- | ---- | ---- |
| L2 (Weight Decay) | $R(W) = \sum_{k,l} W_{k,l}^2$ | 偏好小而分散的权重 |
| L1 | $R(W) = \sum_{k,l} \|W_{k,l}\|$ | 偏好稀疏权重（很多权重为 0） |
| Elastic Net | $R(W) = \beta_1 \sum \|W\| + \beta_2 \sum W^2$ | L1 和 L2 的混合 |

> **L2 正则化的直觉**：
>
> 假设 $x = [1, 1, 1, 1]$，有两组权重：
> - $w_1 = [1, 0, 0, 0]$，点积 = $1$
> - $w_2 = [0.25, 0.25, 0.25, 0.25]$，点积 = $1$
>
> 两者分数相同，但 L2 正则化偏好 $w_2$（$\|w_2\|^2 = 0.25 < \|w_1\|^2 = 1$），因为它让每个特征都"参与"，而不是过度依赖某个特征。

---

## 💡 重点总结

1. **参数化思想**：用 $f(x, W) = Wx + b$ 将"知识"压缩进权重矩阵，预测只需一次矩阵乘法
2. **三种视角**：代数（矩阵运算）、视觉（模板匹配）、几何（超平面划分）
3. **线性分类器的局限**：只能学习线性决策边界，无法处理 XOR 等非线性问题
4. **SVM Loss**：只关心 margin，满足后不再优化
5. **Softmax Loss**：输出概率分布，永远在优化
6. **数值稳定性**：Softmax 必须先减最大值，否则 exp 溢出
7. **正则化**：L2 偏好分散的权重，L1 偏好稀疏的权重
