# Lecture 8：CNN 架构（一）

---

## 📌 概览

本讲回顾 ImageNet 竞赛驱动下的经典 CNN 架构演进：

1. **AlexNet** (2012) — 深度学习的开山之作
2. **VGG** (2014) — 用小滤波器堆叠构建深层网络
3. **GoogLeNet** (2014) — Inception 模块，多尺度特征
4. **ResNet** (2015) — 残差连接，突破深度瓶颈

---

## 第一部分：ImageNet 与架构演进

### 1.1 ImageNet Large Scale Visual Recognition Challenge (ILSVRC)

| 年份 | 冠军 | Top-5 Error | 层数 |
| ---- | ---- | ----------- | ---- |
| 2011 | 传统方法 | 25.7% | — |
| 2012 | **AlexNet** | **16.4%** | 8 |
| 2014 | **VGG / GoogLeNet** | **6.7%** | 19 / 22 |
| 2015 | **ResNet** | **3.6%** | 152 |
| 人类 | — | ~5.1% | — |

> 从 25.7% 到 3.6%，4 年间错误率降低了 **7 倍**。2015 年 ResNet 首次超越人类水平。

## 第二部分：AlexNet (2012)（⭐ 里程碑）

### 2.1 历史意义

- 第一个在 ImageNet 上大幅领先传统方法的深度学习模型
- 证明了 GPU 训练 + 大数据 + 深度网络 = 革命性性能提升
- 开启了深度学习在 CV 领域的统治时代

### 2.2 网络结构

```
Input (227×227×3)
→ Conv1: 96 filters, 11×11, stride 4  → 55×55×96
→ MaxPool: 3×3, stride 2              → 27×27×96
→ Conv2: 256 filters, 5×5, pad 2      → 27×27×256
→ MaxPool: 3×3, stride 2              → 13×13×256
→ Conv3: 384 filters, 3×3, pad 1      → 13×13×384
→ Conv4: 384 filters, 3×3, pad 1      → 13×13×384
→ Conv5: 256 filters, 3×3, pad 1      → 13×13×256
→ MaxPool: 3×3, stride 2              → 6×6×256
→ FC6: 4096
→ FC7: 4096
→ FC8: 1000 (类别数)
```

### 2.3 AlexNet 的关键创新

| 创新 | 说明 |
| ---- | ---- |
| ReLU 激活 | 代替 Sigmoid/Tanh，解决梯度消失，加速训练 |
| GPU 训练 | 使用 2 块 GTX 580，模型拆分到两块 GPU 上 |
| Dropout | 在 FC 层使用 Dropout（p=0.5）防止过拟合 |
| 数据增强 | 随机裁剪、水平翻转、颜色抖动 |
| Local Response Normalization | 局部响应归一化（后来被 BN 取代） |

## 第三部分：VGG (2014)（⭐ 设计哲学）

### 3.1 核心思想

> **"用小滤波器（3×3）堆叠代替大滤波器"**

### 3.2 为什么 3×3 更好？（⭐ 经典面试题）

**两层 3×3 卷积** 的感受野 = **一层 5×5 卷积** 的感受野
**三层 3×3 卷积** 的感受野 = **一层 7×7 卷积** 的感受野

但参数量更少，非线性更多：

| 方案 | 感受野 | 参数量（单通道） | 非线性层数 |
| ---- | ------ | ---------------- | ---------- |
| 1 × (7×7) | 7×7 | $7 \times 7 = 49$ | 1 |
| 3 × (3×3) | 7×7 | $3 \times (3 \times 3) = 27$ | 3 |

> **更少的参数 + 更多的非线性 = 更强的表达能力**

### 3.3 VGG-16 架构

```
[Conv3-64]  × 2 → Pool    (224→112)
[Conv3-128] × 2 → Pool    (112→56)
[Conv3-256] × 3 → Pool    (56→28)
[Conv3-512] × 3 → Pool    (28→14)
[Conv3-512] × 3 → Pool    (14→7)
FC-4096 → FC-4096 → FC-1000
```

- 总参数：~138M（绝大部分在 FC 层）
- 所有卷积都是 3×3, stride 1, pad 1
- 所有池化都是 2×2, stride 2

### 3.4 VGG 的问题

| 问题 | 说明 |
| ---- | ---- |
| 参数量巨大 | 138M 参数，主要在 FC 层 |
| 计算量大 | 推理一张图需要 ~15.5G FLOPs |
| 内存占用高 | 前几层特征图非常大 |

## 第四部分：GoogLeNet / Inception (2014)

### 4.1 核心思想：Inception 模块

在同一层同时使用**不同大小**的滤波器，然后拼接输出：

```
            输入
      ┌──────┼──────┐──────┐
    1×1    3×3    5×5    Pool
    Conv   Conv   Conv   3×3
      └──────┼──────┘──────┘
         拼接 (concat)
            输出
```

### 4.2 1×1 卷积的"瓶颈"作用（⭐ 重要概念）

5×5 卷积计算量很大。解决方案：**先用 1×1 卷积降低通道数，再用大卷积**。

| 方案 | 运算量 |
| ---- | ------ |
| 直接 5×5 Conv（192→32） | 120M ops |
| 1×1 Conv（192→16）→ 5×5 Conv（16→32） | 12.4M ops |

> 1×1 卷积作为"瓶颈层 (Bottleneck)"，减少了 **10 倍**计算量！

### 4.3 GoogLeNet 特点

- 22 层深，但只有 **5M** 参数（VGG 的 1/27）
- 没有 FC 层，使用 **Global Average Pooling**
- 使用辅助分类器 (Auxiliary Classifiers) 帮助训练

## 第五部分：ResNet (2015)（⭐⭐ 最重要的架构）

### 5.1 深度困境

实验发现：简单地堆叠更多层，准确率反而**下降**了（不是过拟合，而是训练误差也更高）。

> 56 层网络的训练误差比 20 层的还高！理论上更深的网络至少不应该更差（可以把多出来的层学成恒等映射）。

### 5.2 残差连接 (Residual Connection / Skip Connection)（⭐⭐ 核心创新）

$$\text{output} = F(x) + x$$

- $x$：输入（直接跳过去）
- $F(x)$：残差函数（需要学习的部分）
- 网络只需要学习 $F(x) = \text{output} - x$，即"**残差**"

```
x ──────────────────(+)──→ output
  │                  ↑
  └→ [Conv→BN→ReLU→Conv→BN] → F(x)
        残差分支
```

### 5.3 为什么残差连接有效？（⭐ 深层理解）

| 角度 | 解释 |
| ---- | ---- |
| 学习难度 | 学习 $F(x) = 0$（恒等映射）比学习 $H(x) = x$ 更容易 |
| 梯度流动 | 梯度可以通过 skip connection 直接回传，缓解梯度消失 |
| 集成效应 | ResNet 可以看作许多不同深度路径的隐式集成 |

### 5.4 残差块的两种形式

**基本残差块** (Basic Block, 用于 ResNet-18/34)：

```
x → [3×3 Conv → BN → ReLU → 3×3 Conv → BN] → (+x) → ReLU
```

**瓶颈残差块** (Bottleneck Block, 用于 ResNet-50/101/152)：

```
x → [1×1 Conv → BN → ReLU → 3×3 Conv → BN → ReLU → 1×1 Conv → BN] → (+x) → ReLU
       降维                      卷积                      升维
```

1×1 卷积先降通道数（如 256→64），3×3 卷积在低维空间操作，再 1×1 升回来——**大幅减少计算量**。

### 5.5 ResNet 家族

| 模型 | 层数 | 参数量 | Top-5 Error |
| ---- | ---- | ------ | ----------- |
| ResNet-18 | 18 | 11.7M | ~10.9% |
| ResNet-34 | 34 | 21.8M | ~8.6% |
| ResNet-50 | 50 | 25.6M | ~7.1% |
| ResNet-101 | 101 | 44.5M | ~6.4% |
| ResNet-152 | 152 | 60.2M | ~6.2% |

## 第六部分：模型对比

### 6.1 准确率 vs 计算量

| 模型 | Top-5 Error | 参数量 | FLOPs |
| ---- | ----------- | ------ | ----- |
| AlexNet | 16.4% | 62M | 0.7G |
| VGG-16 | 7.4% | 138M | 15.5G |
| GoogLeNet | 6.7% | 5M | 1.5G |
| ResNet-50 | 6.7% | 25.6M | 3.9G |
| ResNet-152 | 6.2% | 60.2M | 11.5G |

> **GoogLeNet 和 ResNet-50** 在准确率接近的情况下，参数量和计算量远小于 VGG。

---

## 💡 重点总结

1. **AlexNet**：开启深度学习时代，关键创新是 ReLU + GPU 训练 + Dropout
2. **VGG**：3×3 小滤波器堆叠 > 大滤波器，更少参数更多非线性
3. **GoogLeNet**：Inception 多尺度特征 + 1×1 瓶颈降维，5M 参数击败 138M 的 VGG
4. **ResNet**：残差连接是最重要的架构创新，使训练 100+ 层成为可能
5. **瓶颈设计**：1×1 Conv 降维 → 3×3 Conv → 1×1 Conv 升维，大幅节省计算
6. **趋势**：网络越来越深，但通过巧妙设计控制参数量和计算量
