## 第一部分：激活函数（⭐⭐ 面试高频考点）

### 1.1 Sigmoid

$$\sigma(x) = \frac{1}{1 + e^{-x}}$$

输出范围：$(0, 1)$

**三大致命问题**：

| 问题 | 说明 |
| ---- | ---- |
| ① 梯度消失 | 当 $x$ 很大或很小时，$\sigma'(x) \approx 0$，梯度几乎为零 |
| ② 非零中心输出 | 输出总是正数，导致梯度要么全正要么全负（锯齿形更新） |
| ③ 指数运算开销 | $e^{-x}$ 的计算比 ReLU 慢 |

> **梯度消失详解**：$\sigma'(x) = \sigma(x)(1 - \sigma(x))$，最大值只有 $0.25$。经过多层之后梯度呈指数级衰减。

### 1.2 Tanh

$$\tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}$$

输出范围：$(-1, 1)$

- ✅ 解决了 Sigmoid 的"非零中心"问题
- ❌ 仍然有梯度消失问题（饱和区梯度 ≈ 0）

### 1.3 ReLU（⭐ 最常用）

$$\text{ReLU}(x) = \max(0, x)$$

| 优点 | 说明 |
| ---- | ---- |
| 计算高效 | 只需要一次比较操作 |
| 不饱和 | 正区间梯度恒为 1，不会梯度消失 |
| 收敛快 | 实验中比 Sigmoid/Tanh 快 ~6 倍 |
| 稀疏激活 | 很多神经元输出 0，天然稀疏 |

**ReLU 的问题——"死神经元" (Dead ReLU)**：

当 $x < 0$ 时梯度为 0。如果某个神经元的输入总是负数，梯度永远为 0，权重永远不更新——这个神经元就"死"了。

> **经验**：使用较小的正偏置 ($b = 0.01$) 或 Leaky ReLU 来缓解。

### 1.4 Leaky ReLU

$$f(x) = \begin{cases} x & x > 0 \\ 0.01x & x \leq 0 \end{cases}$$

- 负区间也有小梯度（$0.01$），避免死神经元
- PReLU：负区间斜率是**可学习**的参数 $\alpha$

### 1.5 ELU (Exponential Linear Unit)

$$f(x) = \begin{cases} x & x > 0 \\ \alpha(e^x - 1) & x \leq 0 \end{cases}$$

- 输出均值更接近 0（比 ReLU 好）
- 负区间有梯度且饱和（对噪声更鲁棒）
- 但有指数运算开销

### 1.6 GELU / Swish（现代选择）

$$\text{GELU}(x) = x \cdot \Phi(x)$$

$$\text{Swish}(x) = x \cdot \sigma(\beta x)$$

- 在 Transformer / 现代架构中表现优异
- 处处平滑，适合优化

### 1.7 激活函数选择建议（⭐ 实战经验）

```
优先选择：ReLU
如果死神经元严重：Leaky ReLU / ELU
Transformer 中：GELU
不要使用：Sigmoid / Tanh（除了特殊场景如门控机制）
```

## 第二部分：数据预处理

### 2.1 为什么要预处理？

如果输入特征的尺度差异很大（比如一个维度范围 [0, 1000]，另一个 [0, 1]），梯度在不同方向上的大小会差异巨大，导致优化路径"锯齿形"，收敛很慢。

### 2.2 常见预处理方法

#### 零均值化 (Zero-centering)

$$x' = x - \mu$$

让数据以原点为中心。$\mu$ 是训练集的均值。

#### 标准化 (Normalization)

$$x' = \frac{x - \mu}{\sigma}$$

让每个维度的均值为 0，标准差为 1。

### 2.3 图像领域的实践（⭐ 工程经验）

| 方法 | 说明 | 使用场景 |
| ---- | ---- | -------- |
| 减去均值图像 | 对每个像素位置减去训练集的均值 | AlexNet |
| 减去通道均值 | 对 R/G/B 三个通道分别减去均值 | VGG, ResNet |
| 标准化到 [0,1] | 除以 255 | 常见的默认做法 |

> **关键原则**：预处理的统计量（均值、标准差）必须在**训练集**上计算，然后**同样应用**到验证集和测试集。

### 2.4 图像一般不做 PCA / 白化

- 图像像素之间的空间关系很重要，PCA 会破坏这种结构
- CNN 可以自己学习需要的变换

## 第三部分：权重初始化（⭐⭐ 极其重要）

### 3.1 全零初始化 —— ❌ 灾难性错误

如果所有权重都初始化为 0：
- 所有神经元的输出相同
- 所有梯度相同
- 所有权重更新相同
- 网络永远**无法打破对称性**，等价于只有一个神经元

### 3.2 随机小值初始化 —— ⚠️ 有问题

```python
W = 0.01 * torch.randn(D_in, D_out)
```

- 对浅层网络可以工作
- 对深层网络：随着层数增加，激活值会**不断衰减到 0**

### 3.3 问题分析：方差变化

假设输入 $x$ 的每个维度方差为 $\text{Var}(x) = 1$，权重方差为 $\text{Var}(w)$：

$$\text{Var}(y) = \text{Var}\left(\sum_{i=1}^n w_i x_i\right) = n \cdot \text{Var}(w) \cdot \text{Var}(x)$$

- 如果 $\text{Var}(w)$ 太小：$\text{Var}(y) \to 0$，激活值萎缩（**梯度消失**）
- 如果 $\text{Var}(w)$ 太大：$\text{Var}(y) \to \infty$，激活值爆炸（**梯度爆炸**）

### 3.4 Xavier 初始化（⭐ 适用于 Sigmoid/Tanh）

$$W \sim \mathcal{N}\left(0, \frac{1}{n_{\text{in}}}\right)$$

或等价地：

$$W \sim U\left(-\sqrt{\frac{1}{n_{\text{in}}}}, \sqrt{\frac{1}{n_{\text{in}}}}\right)$$

**推导思路**：要保持每层输出的方差不变，需要 $n_{\text{in}} \cdot \text{Var}(w) = 1$，即 $\text{Var}(w) = \frac{1}{n_{\text{in}}}$。

### 3.5 Kaiming 初始化（⭐ 适用于 ReLU）

Xavier 初始化假设激活函数是线性的。但 ReLU 会把一半的值变成 0，方差减半。所以需要补偿：

$$W \sim \mathcal{N}\left(0, \frac{2}{n_{\text{in}}}\right)$$

多了一个系数 $2$，因为 ReLU 让方差减半。

### 3.6 初始化总结

| 初始化方法 | 适用激活函数 | 方差 |
| ---------- | ------------ | ---- |
| Xavier | Sigmoid, Tanh | $\frac{1}{n_{\text{in}}}$ |
| Kaiming (He) | ReLU, Leaky ReLU | $\frac{2}{n_{\text{in}}}$ |

> **实践建议**：用 ReLU 就用 Kaiming 初始化，用 Tanh 就用 Xavier。PyTorch 默认的 `nn.Linear` 和 `nn.Conv2d` 已经内置了合理的初始化。

### 3.7 有了 BN 还需要好的初始化吗？

BN 会归一化每层的激活值，一定程度上缓解了初始化的影响。但**好的初始化仍然有帮助**：

- 加速训练初期的收敛
- 没有 BN 的场景（如 RNN）仍然依赖初始化

---

## 💡 重点总结

1. **ReLU 是默认选择**：简单、高效、不饱和，但要注意死神经元
2. **Sigmoid 不要用**：梯度消失 + 非零中心，深度网络的噩梦
3. **数据预处理**：至少做零均值化，统计量来自训练集
4. **全零初始化是致命错误**：对称性无法打破
5. **Xavier 初始化**：适配 Sigmoid/Tanh，$\text{Var} = 1/n_{\text{in}}$
6. **Kaiming 初始化**：适配 ReLU，$\text{Var} = 2/n_{\text{in}}$（补偿 ReLU 的方差减半）
7. **初始化的本质**：让每层激活值的方差保持稳定，防止梯度消失/爆炸
