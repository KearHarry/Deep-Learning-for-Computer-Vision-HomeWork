## 第一部分：学习率调度 (Learning Rate Schedules)

### 1.1 为什么需要调度？

固定学习率的问题：
- 太大 → 后期在最优解附近震荡，无法收敛
- 太小 → 训练极慢，可能困在局部最优

**解决方案**：先用大学习率快速下降，后用小学习率精细调整。

### 1.2 常见学习率调度策略

#### Step Decay

$$\text{lr}_t = \text{lr}_0 \times \gamma^{\lfloor t / T \rfloor}$$

每经过 $T$ 个 epoch，学习率乘以衰减因子 $\gamma$（通常 $\gamma = 0.1$）。

> ResNet 论文的做法：初始 lr=0.1，在第 30、60、90 个 epoch 各除以 10。

#### Cosine Annealing（⭐ 现代常用）

$$\text{lr}_t = \frac{\text{lr}_0}{2}\left(1 + \cos\left(\frac{t \pi}{T}\right)\right)$$

学习率从初始值平滑下降到接近零，曲线如半个余弦波。

#### Linear Warmup + Cosine Decay（⭐⭐ 最佳实践）

```
lr
 ↑
 |  /‾‾‾‾‾‾‾\
 | /          ‾‾‾‾‾‾‾‾‾\
 |/                       ‾‾‾→
 +------|-------------------→ epoch
    warmup     cosine decay
```

前几个 epoch 线性增加学习率（warmup），然后用余弦退火下降。

**为什么需要 Warmup？**
- 训练初期，模型参数随机，梯度方向不稳定
- 大学习率 + 不稳定梯度 = 训练发散
- Warmup 让模型先"热身"，等参数稍微有意义了再加大步伐

## 第二部分：超参数搜索

### 2.1 搜索策略

| 策略 | 说明 | 推荐 |
| ---- | ---- | ---- |
| 网格搜索 (Grid Search) | 均匀采样每个超参数的所有组合 | ❌ 维度灾难 |
| 随机搜索 (Random Search) | 每个超参数独立随机采样 | ✅ 更高效 |

**随机搜索更好的原因**：如果某个超参数比其他更重要，随机搜索能在那个维度上探索更多不同的值。

### 2.2 搜索流程（⭐ 实践经验）

```
第一轮：粗搜索（大范围，少 epoch）
  → 学习率: [1e-5, 1e-1]（对数尺度）
  → 正则化: [1e-6, 1e-2]
  → 每个配置只跑 5 个 epoch

第二轮：精搜索（缩小范围，多 epoch）
  → 根据第一轮结果缩小范围
  → 每个配置跑完整训练

诊断：
  → 如果 loss 不下降：学习率太低
  → 如果 loss 爆炸/NaN：学习率太高
  → 如果训练 acc ≫ 验证 acc：过拟合，加正则化
  → 如果训练 acc ≈ 验证 acc 且都低：欠拟合，增大模型
```

### 2.3 学习率是最重要的超参数

> **经验法则**：如果只能调一个超参数，调学习率。

## 第三部分：数据增强 (Data Augmentation)（⭐ 重要）

### 3.1 核心思想

在训练时对输入图像施加随机变换，**人为增加训练数据的多样性**，而不改变标签。

### 3.2 常见增强方法

| 方法 | 说明 |
| ---- | ---- |
| 随机裁剪 (Random Crop) | 从图像中随机裁剪一部分区域 |
| 水平翻转 (Horizontal Flip) | 以 50% 概率左右翻转 |
| 色彩抖动 (Color Jitter) | 随机调整亮度、对比度、饱和度 |
| 随机擦除 (Random Erasing) | 随机遮挡图像的一个矩形区域 |
| Mixup | 两张图和标签的线性插值 |
| CutMix | 从另一张图裁剪区域贴到当前图 |

### 3.3 注意事项

- 数据增强只在**训练时**使用
- 测试时使用固定的处理（如中心裁剪）
- 增强不应改变图像的语义（如垂直翻转文字图像就不合适）

## 第四部分：迁移学习 (Transfer Learning)（⭐⭐ 极其重要）

### 4.1 核心思想

用在大数据集（如 ImageNet）上预训练的模型，迁移到自己的小数据集任务上。

### 4.2 迁移方式

| 数据量 | 与预训练数据相似度 | 策略 |
| ------ | ------------------- | ---- |
| 少 | 高 | 冻结所有层，只训练最后的分类器 |
| 少 | 低 | 冻结前面层，微调后面几层 |
| 多 | 高 | 微调整个网络（用较小学习率） |
| 多 | 低 | 微调整个网络或从头训练 |

### 4.3 为什么迁移学习有效？

CNN 的底层（前几层）学到的是通用特征：边缘、角点、纹理。这些对所有视觉任务都有用。高层才学到任务特定的语义特征。

> **实践建议**：几乎所有的计算机视觉项目都应该从预训练模型开始，除非数据量极其庞大。

---

## 💡 重点总结

1. **Cosine Annealing + Linear Warmup**：现代训练的标准学习率调度方案
2. **随机搜索优于网格搜索**：更高效地利用搜索预算
3. **学习率是最重要的超参数**：先调它，其他的影响都不如它大
4. **数据增强是免费的午餐**：不需要更多真实数据就能提升泛化
5. **迁移学习是默认选择**：小数据集上从头训练几乎总是不如微调预训练模型
