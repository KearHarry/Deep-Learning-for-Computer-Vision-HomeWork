## 📌 概览

2. **深度估计** — 从单张图像推断深度
3. **3D 目标检测** — 在 3D 空间中定位物体

## 第一部分：3D 数据表示

### 1.1 主要表示方式

| 表示 | 形式 | 优点 | 缺点 |
| ---- | ---- | ---- | ---- |
| 深度图 (Depth Map) | 2D 图像，每像素一个深度值 | 与图像对齐 | 只有单视角 |
| 体素 (Voxel) | 3D 网格，每格占用/空 | 规则结构，适合 3D CNN | 内存随分辨率立方增长 |
| 点云 (Point Cloud) | 无序的 3D 点集 $(x, y, z)$ | 灵活、紧凑 | 无序，不适合传统 CNN |
| 三角网格 (Mesh) | 顶点 + 面片 | 适合渲染 | 拓扑复杂 |
| 隐式函数 (Implicit) | $f(x, y, z) = 0$ 的等值面 | 分辨率无关 | 推理慢 |

### 1.2 点云处理：PointNet（⭐ 重要）

**挑战**：点云是**无序集合**，模型输出不应依赖点的输入顺序。

**PointNet 的解决方案**：

```
每个点 (x,y,z)
  → 共享 MLP → 每点特征 (N, D)
  → Max Pooling（跨所有点）→ 全局特征 (D,)
  → MLP → 分类/分割结果
```

- **共享 MLP**：对每个点独立处理（置换等变）
- **Max Pooling**：对称函数，保证置换不变性
- **局限**：忽略了点之间的局部结构

### 1.3 PointNet++

在 PointNet 基础上引入层级结构：

1. **采样**：FPS (Farthest Point Sampling) 选代表点
2. **分组**：Ball Query 找每个代表点的邻域
3. **特征提取**：对每个局部区域用 PointNet

类似 CNN 的层级特征提取，但作用在不规则点云上。

## 第二部分：深度估计

### 2.1 单目深度估计

从**单张 RGB 图像**预测每个像素的深度值。

```
RGB 图像 (3, H, W) → 编码器-解码器网络 → 深度图 (1, H, W)
```

- 本质上是一个回归问题
- 依赖单目线索：纹理梯度、遮挡关系、先验知识
- 是一个"病态问题"（同一张图对应无穷多种 3D 场景）

### 2.2 立体视觉 (Stereo Vision)

用**两台相机**（左右视图）估计深度，基于三角测量原理：

$$\text{depth} = \frac{f \cdot B}{d}$$

- $f$：焦距，$B$：基线（两相机距离），$d$：视差

## 第三部分：3D 目标检测

### 3.1 输入类型

- **纯点云**：PointPillars、VoxelNet
- **图像 + 点云**：多模态融合
- **纯图像**：单目 3D 检测（较难）

### 3.2 与 2D 检测的区别

| | 2D 检测 | 3D 检测 |
| -- | ------- | ------- |
| 框 | 4 参数 $(x_1, y_1, x_2, y_2)$ | 7+ 参数 $(x, y, z, w, h, l, \theta)$ |
| 输入 | 图像 | 点云/图像/融合 |
| 尺度 | 像素 | 米 |
| 应用 | 通用 | 自动驾驶为主 |

---

## 💡 重点总结

1. **3D 有多种数据表示**，各有优劣，选择取决于应用场景
2. **PointNet 用共享 MLP + Max Pooling** 解决点云的无序性问题
3. **单目深度估计**是病态问题，需要依赖学习到的先验知识
4. **3D 检测需要预测更多参数**（位置、尺寸、朝向），比 2D 检测难得多
5. **体素表示受限于内存**（$O(N^3)$），隐式表示是突破分辨率限制的方向
