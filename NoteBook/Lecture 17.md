# Lecture 17：视频理解 (Video Understanding)

---

## 📌 概览

本讲将视觉理解从静态图像扩展到动态视频：

1. **视频分类** — 判断视频中的动作/活动
2. **时空建模** — 如何同时处理空间和时间信息
3. **主流架构** — 3D CNN、Two-Stream、SlowFast

---

## 第一部分：视频理解的挑战

### 1.1 视频 vs 图像

| 维度 | 图像 | 视频 |
| ---- | ---- | ---- |
| 数据形状 | (C, H, W) | (C, T, H, W) |
| 信息维度 | 空间 | 空间 + 时间 |
| 数据量 | 1 帧 | 30-60 帧/秒 × 秒数 |
| 计算量 | 中等 | 巨大 |

### 1.2 核心问题

- 如何高效处理时间维度？
- 如何捕捉运动信息（而不仅仅是外观）？
- 如何在不爆显存的情况下处理长视频？

---

## 第二部分：视频分类方法

### 2.1 Late Fusion：逐帧 CNN + 聚合

```
帧1 → CNN → 特征1 ┐
帧2 → CNN → 特征2 ├→ Average/Max Pool → 分类
帧3 → CNN → 特征3 ┘
```

- 简单但忽略了帧间的时间关系
- 无法捕捉运动信息

### 2.2 Early Fusion：3D 卷积

将时间维度也纳入卷积操作：

```
标准 2D 卷积核: (C_in, K_H, K_W)
3D 卷积核:     (C_in, K_T, K_H, K_W)
```

- **C3D**：直接在视频体上做 3×3×3 卷积
- 能学习时空特征，但计算量大
- **I3D (Inflated 3D)**：将预训练的 2D CNN 权重"膨胀"为 3D

### 2.3 Two-Stream Networks（⭐ 经典）

```
空间流：单帧 RGB 图像 → 2D CNN → 外观特征
时间流：堆叠光流 → 2D CNN → 运动特征
→ 两个流的预测融合（平均/拼接）
```

- 空间流捕捉"是什么"（外观）
- 时间流捕捉"怎么动"（运动）
- 光流 (Optical Flow) 显式编码相邻帧的像素位移

### 2.4 SlowFast Networks（⭐ 现代方法）

```
Slow pathway: 低帧率（如 4 fps），大通道数 → 捕捉语义
Fast pathway: 高帧率（如 32 fps），小通道数 → 捕捉运动
→ 横向连接融合两个路径
```

**设计直觉**：
- 语义信息变化慢（一个动作持续好几帧），不需要高帧率
- 运动信息变化快（细微的手部动作），需要高帧率但不需要很多通道

### 2.5 Video Transformers

将视频 patch 序列化输入 Transformer：
- **ViViT**：将视频拆分为时空 patch (tubelet)
- **TimeSformer**：分别在时间和空间维度做注意力（减少计算量）

---

## 💡 重点总结

1. **视频 = 图像 + 时间维度**，核心挑战是高效建模时空关系
2. **3D 卷积**直接在时空体上操作，但计算成本高
3. **Two-Stream** 分离外观和运动，光流提供显式运动信息
4. **SlowFast** 用不同帧率处理语义和运动，效率与性能兼顾
5. **Video Transformer** 是最新趋势，将自注意力扩展到时空域
